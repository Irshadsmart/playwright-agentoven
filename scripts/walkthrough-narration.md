# AgentOven Automation Demo — Narration Script

Paste each block into a text-to-speech tool (e.g. ElevenLabs, Azure TTS, Google TTS, or
read it aloud while recording with OBS).  Each block matches one step in walkthrough.js.

---

## INTRO (before you start the script)

"Welcome to the AgentOven Automation Demo.
In this walkthrough, we will explore every section of the AgentOven AI platform
and then review the automated test results generated by our Playwright test suite.
Let's get started."

---

## Step 01 — Load AgentOven

"We begin by opening AgentOven in the browser at localhost port 8085.
AgentOven is an AI agent orchestration platform that lets you build, manage,
and deploy intelligent agents powered by large language models."

---

## Step 02 — Overview

"The Overview dashboard is the home screen.
It gives a high-level snapshot of the platform — system health,
recent agent activity, and key metrics — all visible at a glance."

---

## Step 03 — Agents — All 5 Cards

"Next we navigate to the Agents section.
Here we can see all five registered AI agents:
My First Agent, task-planner, doc-researcher, summarizer, and quality-reviewer.
Each card shows the agent name, its current status, and action buttons."

---

## Step 04 — Integrate — My First Agent — Invoke Tab

"We click Integrate on My First Agent to open its integration modal.
The Invoke tab shows how to call this agent programmatically via HTTP —
including the endpoint URL, the request body format, and the required auth headers."

---

## Step 05 — Integrate — My First Agent — Session Tab

"Clicking the Session tab shows how to make stateful session calls.
Sessions allow the agent to maintain conversation context across multiple requests —
useful for multi-turn interactions and chatbot-style workflows."

---

## Step 06 — Integrate — My First Agent — Test Tab

"The Test tab provides a live console.
You can type a message directly and see how the agent responds in real time —
without writing any code. It's ideal for quick validation during development."

---

## Step 07 — Integrate — My First Agent — Agent Card Tab

"Finally, the Agent Card tab shows a shareable metadata card.
It describes this agent's purpose, capabilities, and version in a standardised format —
ready to share with other teams or external systems."

---

## Step 08 — Integrate — task-planner

"We now open the Integrate modal for the task-planner agent.
This agent is configured as a QA Test Planner — it breaks down features
into structured test plans. The Invoke tab shows its API integration details."

---

## Step 09 — Integrate — doc-researcher

"Moving to doc-researcher — this agent is configured as a QA Test Case Generator.
It researches documents and generates detailed test cases from them.
The endpoint is the same pattern as the other agents."

---

## Step 10 — Integrate — summarizer

"The summarizer agent takes long content and distills it into clear, concise summaries.
Here we see its invoke endpoint and the expected request format."

---

## Step 11 — Integrate — quality-reviewer

"The quality-reviewer agent checks correctness, clarity, and overall quality of outputs.
It acts as a final validation step in automated pipelines."

---

## Step 12 — Re-cook Action

"Back on the Agents page, we demonstrate the Re-cook action.
Re-cook regenerates an agent's configuration from scratch — using its system prompt
and skills as the source of truth. This is useful after editing the agent definition."

---

## Step 13 — Cool (Deactivate)

"The Cool button deactivates an agent — it stops accepting requests immediately.
This is useful for maintenance windows or when you need to temporarily pause an agent
without deleting it."

---

## Step 14 — Rewarm (Reactivate)

"Rewarm brings the agent back online.
The agent is now live again and accepting requests.
Cool and Rewarm give you fine-grained control over each agent's availability."

---

## Step 15 — Recipes

"The Recipes section is where we define multi-step workflows.
A recipe chains multiple agents together into an automated pipeline —
for example, plan, research, summarize, then review — all in one sequence."

---

## Step 16 — Create New Recipe

"We create a new recipe named My First Recipe.
We fill in a name and a short description, then click Create.
The recipe is saved and ready to be configured with agent steps."

---

## Step 17 — Run Recipe

"Clicking Run executes the recipe immediately.
It triggers the full agent pipeline defined in this workflow.
The results will be stored in DishShelf."

---

## Step 18 — DishShelf

"DishShelf is the output store for completed recipe runs.
Every time a recipe is executed, the result is saved here —
providing a persistent record of all pipeline outputs."

---

## Step 19 — DishShelf → Recipes Navigation

"Each item on DishShelf links back to the recipe that produced it.
Clicking the Recipes link takes us back to the source —
giving full traceability from output to origin."

---

## Step 20 — Prompts

"The Prompts section stores reusable prompt templates.
We have two prompts configured: qa-test-generator and code-reviewer.
These templates are shared across agents to ensure consistency."

---

## Step 21 — Edit qa-test-generator

"Opening the Edit form for qa-test-generator, we can see its Tags —
testing, qa, automation — and the full Template text that is sent to the language model.
Centralising prompts here means any agent can reuse and benefit from them."

---

## Step 22 — Edit code-reviewer

"The code-reviewer prompt template guides the agent through a systematic code review.
Tags include code, review, engineering, and best-practices.
The template structures the agent's response for maximum consistency."

---

## Step 23 — Providers

"The Providers section shows the configured LLM backends.
We have two providers: My Anthropic, using the Claude Sonnet model,
and My OpenAI, using GPT-4o. Both are ready to serve agent requests."

---

## Step 24 — Test OpenAI Connection

"We click Test on the My OpenAI provider to verify the connection is live.
AgentOven sends a lightweight ping to the OpenAI API endpoint
and reports back whether the authentication and connectivity are working."

---

## Step 25 — Test Anthropic Connection

"We do the same for the Anthropic provider.
Both providers are verified — the platform can route requests to either
Claude or GPT-4o depending on the agent configuration."

---

## Step 26 — Model Catalog

"The Model Catalog lists over two thousand AI models available across all providers.
You can browse, filter by provider or capability, and assign specific models
to individual agents or recipes."

---

## Step 27 — MCP Tools

"MCP Tools stands for Model Context Protocol tools.
These are external capabilities — like Weather data and Google Search —
that any agent can call during a run.
Tools extend what agents can do beyond pure language generation."

---

## Step 28 — Traces

"The Traces section is the full audit trail of the platform.
Every agent run, every decision, every tool call is logged here
with timestamps and inputs and outputs — essential for debugging and compliance."

---

## Step 29 — Embeddings

"Embeddings are the bridge between text and vectors.
This section configures which embedding model converts your documents
into numerical representations, ready to be stored in a vector database."

---

## Step 30 — Vector Stores

"Vector Stores are the databases where embeddings live.
When an agent needs to search documents semantically — meaning by meaning,
not just by keyword — it queries the vector store to find the most relevant content."

---

## Step 31 — RAG Pipelines

"RAG stands for Retrieval-Augmented Generation.
This section manages the full pipeline — from ingesting documents
to querying them with AI — so agents can answer questions grounded in real data."

---

## Step 32 — RAG Ingest Tab

"We switch to the Ingest tab to add a document to the vector store.
Here we paste the content we want the AI to be able to retrieve and answer questions about."

---

## Step 33 — Ingest BDD Scenario

"We paste in a BDD test scenario describing the AgentOven system.
When we click Ingest Document, the platform vectorises this text
and stores it — the AI can now answer questions about it using semantic search."

---

## Step 34 — Connectors

"Data Connectors allow external data sources — databases, APIs, file systems —
to be plugged directly into the platform.
This is where you connect AgentOven to your existing data infrastructure."

---

## Step 35 — Automation Report — HTML

"Now let's look at the automated test results.
This is the AgentOven Automation Report — generated automatically after every test run.
It shows all eighteen tests with pass/fail status, execution time, and screenshots.
Cards are collapsed by default — click the arrow to expand any test and see its screenshots."

---

## Step 36 — Report Card Expanded

"Here we expand one test card to reveal the embedded screenshot.
Each screenshot was captured live by the test automation during the run —
exactly what was visible on screen at that moment."

---

## Step 37 — Automation Report — PDF

"This is the PDF version of the same report.
It is ready to email or print — a clean, professional summary
showing the test results and platform status for management review."

---

## Step 38 — Playwright HTML Report

"Finally, the Playwright built-in test report.
This shows the test execution timeline, pass and fail counts,
and links to detailed trace files for debugging any failures."

---

## OUTRO

"That concludes the AgentOven Automation Demo.
We have walked through all thirteen sections of the platform —
from AI agents and recipes to RAG pipelines and connectors —
and reviewed the automated test report generated by our Playwright test suite.
All eighteen tests passed. The platform is healthy and fully operational.
Thank you for watching."
